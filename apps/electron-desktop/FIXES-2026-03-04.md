# Fixes — 2026-03-04

## 1. Chave OpenAI incorreta para embeddings (401 invalid_api_key)

**Sintoma:** O módulo `[memory]` falhava ao fazer sync de embeddings via OpenAI com erro `401 invalid_api_key`. A chave mascarada no log terminava em `...2XfH`, que correspondia a uma chave Groq antiga — não à chave OpenAI.

**Causa raiz:** Uma chave de API anterior (possivelmente Groq ou uma chave OpenAI revogada) estava sendo utilizada pelo serviço de embeddings. O arquivo `auth-profiles.json` em `~/Library/Application Support/atomicbot-desktop/openclaw/agents/main/agent/auth-profiles.json` já continha a chave OpenAI correta (`sk-proj-...6xgA`), indicando que o erro era de uma sessão anterior com credenciais desatualizadas.

**Correção:** Confirmado que `auth-profiles.json` já possui a chave correta. O erro se resolve ao reiniciar o gateway (nova sessão usa o perfil atualizado).

**Arquivo afetado:**

- `~/Library/Application Support/atomicbot-desktop/openclaw/agents/main/agent/auth-profiles.json`

## 2. `rg` (ripgrep) não encontrado

**Sintoma:** `zsh:1: command not found: rg` — o módulo `[tools]` tentava executar `rg` no workspace do agente, mas o binário não existia no sistema.

**Causa raiz:** O ripgrep não estava instalado na máquina. O gateway e os agentes dependem de `rg` para buscas de conteúdo no workspace.

**Correção:** Instalado via Homebrew:

```bash
brew install ripgrep
```

Binário disponível em `/opt/homebrew/bin/rg` (ripgrep 15.1.0).

## 3. `openclaw` CLI e `rg` fora do PATH do processo Electron

**Sintoma:** Mesmo com `rg` instalado e `openclaw` disponível no terminal do usuário, o gateway spawned pelo Electron não encontrava esses binários. Erros `command not found: rg` e `command not found: openclaw`.

**Causa raiz:** O Electron no macOS herda um PATH mínimo (não o PATH completo do shell de login do usuário). O `spawn.ts` passava `process.env.PATH` do processo Electron para o gateway, que não incluía `/opt/homebrew/bin` nem outros diretórios do usuário. Embora o core tenha `ensureOpenClawCliOnPath()` (em `src/infra/path-env.ts`) que adiciona diretórios comuns, e `shell-env.ts` que pode importar variáveis do shell de login, essa funcionalidade só é ativada com `OPENCLAW_LOAD_SHELL_ENV=1`.

**Correção:** Adicionado `OPENCLAW_LOAD_SHELL_ENV: "1"` ao env do processo gateway em `apps/electron-desktop/src/main/gateway/spawn.ts:148`. Isso faz o gateway importar variáveis de ambiente faltantes (incluindo entradas do PATH) do shell de login do usuário.

**Diff:**

```diff
  // apps/electron-desktop/src/main/gateway/spawn.ts
  const env: NodeJS.ProcessEnv = {
    ...process.env,
    OPENCLAW_STATE_DIR: stateDir,
    OPENCLAW_CONFIG_PATH: configPath,
    OPENCLAW_GATEWAY_PORT: String(port),
    OPENCLAW_GATEWAY_TOKEN: token,
    PATH: mergedPath,
    GH_CONFIG_DIR: ghConfigDir,
+   // Allow the gateway to import missing env vars (incl. PATH entries) from the login shell.
+   OPENCLAW_LOAD_SHELL_ENV: "1",
    NO_COLOR: "1",
    FORCE_COLOR: "0",
    OPENCLAW_NO_RESPAWN: "1",
  };
```

**Arquivo afetado:**

- `apps/electron-desktop/src/main/gateway/spawn.ts`
